# Antigravity 体验与 Agent 能力分析

---

## 核心观点

基于最新版 Antigravity（Gemini 驱动）的深度使用体验，对当前 Agent 的能力进行评估分析。

## 评估维度

### 1. 代码生成能力
* 业界对 Gemini 的代码表现评价不一，因未直接测试 3.5 版本，此处不做主观定论，保持观察。

### 2. 系统流畅度与质量
* 软件整体流畅度及质量表现优异，运行过程中未遇到 Bug。
* 响应速度非常快（尤其是其运行的 Gemini 响应速度远快于先前使用过的国产大模型，这一点对体验提升至关重要），且 UI 设计合理、符合直觉。

### 3. 浏览器自动化（核心优势）
* 浏览器操作的优化程度显著，相比数月前的 OpenClaw 体验好得多。
* 从当前的稳定性和交互能力看，已具备直接在生产环境落地的自动化能力。

### 4. Harness 功能
* 引入了 Harness 机制。对习惯于“走一步想一步”的即时探索式工作流用户而言，该功能目前非刚需，但提供了系统化评估的边界能力。

### 5. 研发工作流与 Git 集成
* 对于拥有重度 Git 习惯的开发者而言，当前系统与 Git 的集成仍有欠缺，日常开发和版本管理依然离不开 VS Code。

### 6. 交互设计与上下文习惯
* 这类 Agent 软件普遍倾向于“一个对话解决一个任务”的单点流，这与开发者的实际习惯有冲突。开发者更习惯于通过“Git 管理代码物理状态 + 单个长对话沉淀长上下文 + 多个长对话并行处理不同模块”的方式工作，目前的对话切割方式容易导致上下文连续性被打断，产生一定的使用混乱。

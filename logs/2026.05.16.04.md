# 研究 PR#42631 IR Op 优先级优化
## 1. `set_forward_context` 的作用和运行生命周期

### 作用

`set_forward_context` 是一个**上下文管理器**，它在每次模型推理（forward pass）时被调用，主要做以下几件事：

1. **设置注意力元数据** (`attn_metadata`)：包含 KV cache 的 block 表、序列长度等信息
2. **设置 IR op 优先级**：通过 `vllm_config.kernel_config.ir_op_priority.set_priority()` 设置算子调度优先级
3. **设置 torch wrap 标志**：通过 `vllm.ir.enable_torch_wrap()` 控制是否启用 Torch 算子包装
4. **批次大小统计**：可选地跟踪不同 batch size 的性能数据

### 运行生命周期

```
请求进入
    ↓
Scheduler 调度批次
    ↓
set_forward_context()  ← 每次 forward 都调用
    ↓
模型 forward() 执行推理
    ↓
退出上下文（恢复状态）
    ↓
处理下一个批次...
```

**关键点**：`set_forward_context` 是**每次推理都执行**的，如果一个服务处理 1000 个请求，这个方法就被调用 1000 次。
